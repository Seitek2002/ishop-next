# Доставка: время и гибкая стоимость на бэке

**Кому:** Кума (backend)
**Статус:** ✅ реализовано (бэк + фронт), коммит фронта `0413731`

## Итоговое решение (формат времени)

Выбран **простой строковый `deliveryTime`** — бэк формирует готовую строку сам, фронт только рендерит:
- `delivery_time` заполнен → возвращается как есть (напр. `"1–2 дня"`);
- `delivery_time` пустой, но есть `deliveryTimeMinutes` (напр. `30`) → бэк возвращает `"~30 мин"`;
- оба пустые → `null` (фронт скрывает чип).

Структурный вариант (`min`/`max`/`unit`) отклонён. Фронт **не форматирует** время — `deliveryTimeMinutes` на клиент не прокидывается. Заведения вроде heyyou (`minutes=30`) получают `"~30 мин"` автоматически, без правок в админке.

Тикет про два связанных поля доставки:
1. **Время доставки** (сейчас захардкожено `24–48 ч`).
2. **Гибкая стоимость доставки** (сейчас только число; нужно уметь задавать текст — «Бесплатная», «Оплата за счёт клиента» и т.п.).

## Контекст

В баннере доставки в корзине показывается ориентировочное время доставки. Сейчас оно **захардкожено** на фронте:

```tsx
// src/client-pages/Cart/components/CartUI.tsx:274
<div className='cart__delivery-time ...'>
  <Clock size={14} />
  <span>24–48 ч</span>
</div>
```

То есть для любого заведения всегда отображается «24–48 ч», независимо от реального срока доставки. Это неверно: у разных заведений (и разных режимов — доставка/самовывоз) срок отличается.

Параметры доставки уже частично приходят с бэка — например `deliveryFreeFrom` и `deliveryFixedFee` в `IVenues` (`src/types/venues.types.ts`) из `GET /api/organizations/<slug>/`. Время доставки нужно завести по такому же принципу.

## Что нужно

### Backend

1. Добавить в модель `Organization` поле срока доставки. Варианты формата (на выбор, согласовать с фронтом):
   - простой текст: `deliveryTime: string` (напр. `"24–48 ч"`, `"30–60 мин"`) — максимально гибко, фронт просто рендерит;
   - либо структурно: `deliveryTimeMin: number` + `deliveryTimeMax: number` + `deliveryTimeUnit: 'min' | 'hour'` — тогда фронт сам форматирует и можно локализовать.
   - **Рекомендация:** начать с простого строкового `deliveryTime`, если локализация/расчёты не требуются.
2. Прокинуть поле в ответ `GET /api/organizations/<slug>/` рядом с `deliveryFreeFrom`, `deliveryFixedFee`, `serviceFeePercent`.
3. Дефолт/nullable: если не задано — фронт скроет чип времени (см. ниже), поэтому поле может быть необязательным.

### Frontend (после появления поля на бэке)

1. Добавить в `IVenues` (`src/types/venues.types.ts`):
   ```ts
   deliveryTime?: string | null;
   ```
2. Пробросить значение из `venueData` через `Cart/index.tsx` в `CartUI` (как уже сделано для `deliveryFreeFrom`/`deliveryFee`).
3. В `CartUI.tsx` заменить хардкод `24–48 ч` на значение из пропса.
4. Если `deliveryTime` пустой/`null` — **не показывать** чип времени (а не показывать заглушку).

## Стоимость доставки: гибкий формат (текст вместо только числа)

### Контекст

Сейчас стоимость доставки — это число `deliveryFixedFee` (+ `deliveryFreeFrom` — порог бесплатной доставки). На фронте уже сделано (коммит `ec118f0`): если `deliveryFee === 0`, вместо «0 c» показывается локализованная метка «Бесплатная» (`freeDeliveryLabel`: ru «Бесплатная», kg «Бекер», en «Free»).

Но заведения просят больше гибкости (из отзыва): помимо «Бесплатная» нужно уметь указывать, например, «**Оплата за счёт клиента**», «по договорённости» и т.п. — то есть произвольный текст там, где сейчас только цифра.

### Что нужно (backend)

1. Добавить в модель `Organization` поле текстовой метки стоимости доставки — например `deliveryFeeLabel: string | null` (nullable).
   - Если задано — фронт показывает эту строку как есть (вместо числа): «Бесплатная», «Оплата за счёт клиента», «по договорённости» и т.д.
   - Если пусто — фронт использует обычную логику: `deliveryFixedFee`/`deliveryFreeFrom` (число или авто-метка «Бесплатная» при 0).
2. Прокинуть поле в `GET /api/organizations/<slug>/` рядом с `deliveryFixedFee`/`deliveryFreeFrom`.
3. Согласовать с фронтом: важно решить, **участвует ли** такая «текстовая» доставка в подсчёте итоговой суммы. Скорее всего нет (текст = «считается отдельно/при получении»), тогда `deliveryFee` для итога берётся = 0, а в строке доставки показывается `deliveryFeeLabel`.

### Что нужно (frontend, после бэка)

1. Добавить `deliveryFeeLabel?: string | null` в `IVenues` (`src/types/venues.types.ts`).
2. В `Cart/index.tsx` пробросить значение, и в `CartUI.tsx` (баннер + строка деталей суммы) если `deliveryFeeLabel` задан — показывать его вместо числа/авто-метки.
3. Учесть в подсчёте `total` согласно решению из п.3 выше.

### Acceptance criteria (стоимость)

- [x] Бэк отдаёт `deliveryFeeLabel` в `/api/organizations/<slug>/`.
- [x] В корзине при заданной метке показывается текст (напр. «Оплата за счёт клиента») вместо цифры.
- [x] Поведение итоговой суммы при текстовой доставке согласовано фронт/бэк (текстовая доставка не участвует в `total`).
- [x] Метку можно менять в админке без деплоя фронта.
